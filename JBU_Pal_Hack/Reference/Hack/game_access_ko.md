# 게임 접근 및 변조 분석 (Game Access and Modification Analysis)

## 대상 프로세스
- **프로세스 이름**: `Palworld-Win64-Shipping.exe`
- **아키텍처**: x64

## 메모리 탐색 메커니즘

### 1. AOB (Array of Bytes) 스캐닝
트레이너는 게임 메모리에서 안정적인 코드 주소를 찾기 위해 AOB 스캐닝을 사용합니다. 이를 통해 게임이 업데이트되어 코드 오프셋이 변경되더라도 트레이너가 계속 작동할 수 있습니다.
- **플레이어 스탯 AOB 예시**: `80 BF s1.2 00 00 00 74 * F3 0F 10 * * * * * * * * E8 * * * * * * * * * E8`
- **스텔스 모드 AOB 예시**: `84 C0 74 * * * * * * * * 0F 57 * * * * E8 * * * * 84 C0 74`

### 2. 글로벌 오프셋 테이블 (Global Offsets Table)
트레이너는 팰월드 내부 Unreal Engine 클래스들에 대한 오프셋 목록을 유지합니다. 이 값들은 코드 케이브에서 사용하기 위해 게임의 메모리 문맥에 주입됩니다.

| 심볼 (Symbol) | 오프셋 | 설명 |
| :--- | :--- | :--- |
| `PC_PlayerState` | `0x2B0` | 플레이어 상태 구조체 포인터 |
| `PC_CharacterParameterComponent` | `0x628` | 메인 캐릭터 파라미터 |
| `Param_IndividualParameter` | `0x108` | 개별 스탯 컨테이너 |
| `SParam_HP` | `0x78` | 현재 체력 |
| `SParam_MaxHP` | `0xE8` | 최대 체력 |
| `SParam_FullStomach` | `0x90` | 허기 수치 |
| `SParam_ShieldHP` | `0x108` | 현재 쉴드 |
| `InventoryData_maxInventoryWeight`| `0x164` | 무게 제한 |

## 변조 기법

### 1. 코드 케이브 주입 (Code Cave Injection)
복잡한 변조를 위해 트레이너는 게임 프로세스에 메모리 블록(`newmem`)을 할당하고 게임의 실행 흐름을 이곳으로 돌립니다:
1. AOB 스캔을 통해 대상 명령어를 찾습니다.
2. 해당 명령어를 코드 케이브로 향하는 `jmp` 명령어로 교체합니다.
3. 코드 케이브 내부에서 치트 로직(예: HP를 최대치로 설정)을 수행합니다.
4. 원래의 코드 흐름으로 다시 점프하여 돌아갑니다.

### 2. 직접 명령어 패치 (Direct Instruction Patching)
단순한 치트의 경우 게임 명령어를 직접 수정합니다:
- **예시 (스텔스 모드)**: 감지 체크 후의 조건부 점프(`JZ` - `74`)를 무조건 점프(`JMP` - `EB`)로 변경하여 감지 여부와 상관없이 감지되지 않은 것으로 처리하게 만듭니다.
- **예시 (온도)**: 비교 연산 전에 레지스터를 0으로 초기화하여 환경 온도 효과를 무시하게 만듭니다.
