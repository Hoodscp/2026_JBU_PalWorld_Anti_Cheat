[2-1] FPalIndividualCharacterSaveParameter 추가 멤버 매핑
      (베이스 = UPalIndividualCharacterParameter +0x388)

  FPalIndividualCharacterSaveParameter
    +0x020  uint8         Level              → 절대 0x3A8
    +0x028  int64         Exp                → 절대 0x3B0
    +0x080  uint8         Talent_HP          → 절대 0x408
    +0x081  uint8         Talent_Melee       → 절대 0x409
    +0x082  uint8         Talent_Shot        → 절대 0x40A
    +0x083  uint8         Talent_Defense    → 절대 0x40B
    +0x0A8  FFixedPoint64 MP                 → 절대 0x430
    +0x118  FFixedPoint64 MaxMP              → 절대 0x4A0
    +0x12C  float         SanityValue        → 절대 0x4B4
    +0x2D4  int32         FriendshipPoint    → 절대 0x65C
    +0x2E8  int32         ArenaRankPoint    → 절대 0x670

  → 위 오프셋들은 LocalPlayer 본인 캐릭터 슬롯과 보유 팰 슬롯에
    동일하게 적용된다. PlayerState 체인 / PalStorage 체인 어느 쪽으로
    접근하든 최종 IndividualParameter 베이스 위에 같은 오프셋 테이블
    이 그대로 재사용된다.


[2-2] UPalTechnologyData 미사용 멤버 활용

  UPalTechnologyData
    +0x150  int32 TechnologyPoint     (기존 사용)
    +0x154  int32 BossTechnologyPoint (Offsets.h 에는 이미 정의돼 있었으나
                                      getter/setter 가 미구현 → 본 작업에서
                                      추가)


[2-3] 보유 팰(Pal Storage) 접근 체인

  APalPlayerState
    +0x5E0  UPalPlayerDataPalStorage*  PalStorage

  UPalPlayerDataPalStorage
    +0x030  UPalIndividualCharacterContainer*  TargetContainer

  UPalIndividualCharacterContainer  : UPalContainerBase
    +0x080  TArray<UPalIndividualCharacterSlot*> SlotArray
              { Data(+0x080), Num(+0x088), Max(+0x08C) }

  UPalIndividualCharacterSlot
    +0x060  UPalIndividualCharacterHandle*           Handle
    +0x098  UPalIndividualCharacterParameter*        ReplicateIndividualParameter
            ↑ 이 포인터가 곧 IndividualParameter base 가 되어
              +0x388 (SaveParameter) 임베드 구조가 그대로 따라온다.


[2-4] 최종 보유 팰 포인터 체인

    GameState        (+0x158)
      → PlayerArray  (+0x2A8)
        → PlayerArray[0]  (+0x000)
          → PalStorage              (+0x5E0)
            → TargetContainer       (+0x030)
              → SlotArray           (+0x080)
                → SlotArray[i]      (+0x000)
                  → ReplicateIndividualParameter (+0x098)
                    → SaveParameter base = +0x388
                      → 기존 IndividualParam:: 오프셋 그대로 재사용

[4] 안전성/안티치트 고려
───────────────────────────────────────────────────────────────────────────────
  - 신규 setter 도 기존 패턴 그대로 WriteAt<T>() → IsBadWritePtr 가드 →
    *(T*)addr = value 흐름. 추가적인 raw write 는 없음.
  - 보유 팰 변조는 빈 슬롯에서 nullptr 역참조가 발생하지 않도록
    IsPalSlotEmpty(slot) 로 매 프레임 선검사 + PalParam(slot) 내부에서
    nullptr 시 즉시 return 한다.
  - PalStorage 가 초기에는 0 일 수 있다 (게임 시작 직후 동기화 전).
    이 경우 GetPalSlotCount() == 0 → 메뉴에 "PalStorage not loaded"
    안내 표시되고 Tick 은 무사 무작동.
  - Talent_HP/Melee/Shot/Defense 를 100 까지 채우는 것은 SaveParameter
    가 클라이언트에 임베드된 값이라 즉시 데미지/HP 계수에 반영되지만,
    서버 권위가 있는 모드에선 OnRep_* 으로 되돌려질 수 있다. 검증 후
    필요 시 Track 2 후킹으로 이주 검토.
