[1] 작업 목표
───────────────────────────────────────────────────────────────────────────────
Reference/SDK (Dumper-7 출력본) 를 분석해 다음 게임 데이터 멤버에 대한 read/write
파이프라인을 구축하고 GUI 토글 기반 메모리 치트로 노출한다.

  - UnusedStatusPoint (스킬 포인트 풀)
  - Current HP (HP Value)
  - Stamina (SP)
  - FullStomach (포만감)
  - ShieldHP (실드)
  - TechnologyPoint (연구 포인트)
  - ItemSlotArray / SlotArray[0] / StackCount (인벤토리 첫 슬롯의 개수)
  - CurrentTemperature (온도)  ← 1차 시도 후 본 작업에서는 제외 결정


[2] SDK 구조 분석 결과
───────────────────────────────────────────────────────────────────────────────

[2-1] 핵심 클래스 ↔ 멤버 매핑

  UPalIndividualCharacterParameter
    +0x388  FPalIndividualCharacterSaveParameter SaveParameter (임베드)
            └ 이 베이스(0x388)에 SaveParameter 멤버 오프셋을 더하면
              "절대 오프셋" 이 됨. 기존 코드의 HP=0x400 (=0x388+0x78) 과 정합.

  FPalIndividualCharacterSaveParameter (임베드 구조체)
    +0x078  FFixedPoint64 Hp                  → 절대 0x400 (기존)
    +0x084  float         FullStomach         → 절대 0x40C
    +0x0E0  FFixedPoint64 MaxHP               → 절대 0x468
    +0x100  FFixedPoint64 ShieldHP            → 절대 0x488
    +0x108  FFixedPoint64 ShieldMaxHP         → 절대 0x490
    +0x16C  float         MaxFullStomach      → 절대 0x4F4
    +0x174  uint16        UnusedStatusPoint   → 절대 0x4FC

  UPalCharacterParameterComponent
    +0x130  UPalIndividualCharacterParameter* IndividualParameter (기존)
    +0x328  FFixedPoint64 SP (Stamina, 런타임 전용 — SaveParameter 외부)
    +0x460  UPalItemContainer* ItemContainer (※ 사용하지 않음 — 후술)

  APalPlayerState
    +0x308  APawn*                  PawnPrivate           (기존)
    +0x5D8  UPalPlayerInventoryData* InventoryData
    +0x5E8  UPalTechnologyData*      TechnologyData

  UPalPlayerInventoryData
    +0x190  UPalItemContainerMultiHelper* InventoryMultiHelper

  UPalItemContainerMultiHelper
    +0x038  TArray<UPalItemContainer*> Containers
              { Data(+0x038), Num(+0x040), Max(+0x044) }

  UPalItemContainer (← Containers[0] 가 가리키는 컨테이너)
    +0x070  TArray<UPalItemSlot*> ItemSlotArray
              { Data(+0x070), Num(+0x078), Max(+0x07C) }

  UPalItemSlot
    +0x154  int32 StackCount

  UPalTechnologyData
    +0x150  int32 TechnologyPoint
    +0x154  int32 BossTechnologyPoint


[2-2] 최종 인벤토리 포인터 체인 (사용자 제공 도식 일치)

    GameState        (+0x158)
      → PlayerArray  (+0x2A8)
        → PlayerArray[0]  (+0x000)
          → InventoryData         (+0x5D8)
            → InventoryMultiHelper (+0x190)
              → Containers         (+0x038)
                → Containers[0]    (+0x000)
                  → ItemSlotArray  (+0x070)
                    → SlotArray[0] (+0x000)
                      → StackCount (+0x154)


[2-3] 폐기된 1차 분석 (참고용)

  최초에는 "UPalCharacterParameterComponent +0x460 ItemContainer" 를 단축 경로로
  사용했으나 게임에서 실제 슬롯이 잡히지 않았음. 이는 해당 포인터가 플레이어
  메인 인벤토리가 아닌 별도(equip/transient) 컨테이너를 가리키기 때문.
  → InventoryData 를 통한 정공 체인으로 교체.

[4] 안전성/안티치트 고려
───────────────────────────────────────────────────────────────────────────────
  - 모든 setter 는 IsBadWritePtr 로 가드된 후 *(T*)address = value.
  - 모든 chain step (Step()) 은 IsBadReadPtr 로 보호 + 0 전파.
  - 매 프레임 강제 덮어쓰기 방식이라 서버 권한 검증을 통과하지 못하는
    멀티플레이/공식서버에서는 클라이언트 표시값만 잠시 변하고 재동기화 가능.
    이는 현재 프로젝트(연구/실습 목적) 에서 의도된 동작.
